# Version 1.0.0 Changelog (CampAIgn-RPG-Engine engine)

**Purpose:** Library-first **1.0** release — single `campaign_rpg_engine` package on PyPI, no bundled CLI, no `Session.run_command()`.

**Baseline:** **0.7.2** (`v0.7.2`). Pre-1.0 work preserved on branch `archive/0.7.x`.

**Status:** ✅ **Implemented** — tag **`v1.0.0`** pending PyPI publish.

See also: [MIGRATION-0.7-to-1.0.md](../MIGRATION-0.7-to-1.0.md), [v0.7.2-changelog.md](v0.7.2-changelog.md).

---

## Breaking changes

### Package layout

- Collapsed `src/` into a single **`campaign_rpg_engine/`** package at repo root (flat layout).
- Import only `from campaign_rpg_engine import …` — no `src.*` paths.

### CLI removed

- Deleted `realm` console entry point (`main.py`, stepper, CLI docs).
- Manual GM testing: use [CampAIgn-RPG-Studio](https://github.com/DaveH-Ghost/CampAIgn-RPG-Studio) (separate GitHub repo).

### String command layer removed

- `Session.run_command()` and private `_cmd_*` handlers removed.
- `CommandResult` removed from public API.
- Area string parsers moved to `campaign_rpg_engine.area_edit` (for app-owned dispatch, not public `__all__`).

### Examples trimmed

- Removed `examples/minimal-server/`, `examples/reference_handlers/`, `examples/custom_memory/`.
- Kept `examples/lorebook/`; HTTP/reference handlers live in CampAIgn-RPG-Studio.

### CampAIgn-RPG-Studio external

- GM web app moved to [github.com/DaveH-Ghost/CampAIgn-RPG-Studio](https://github.com/DaveH-Ghost/CampAIgn-RPG-Studio).
- Studio depends on `campaign-rpg-engine>=1.0.0` from PyPI (or local editable during co-dev).

---

## Additions / fixes

- Typed `Session` world API is the only editing surface (`create_object`, `edit_agent`, `create_area`, …).
- Public API surface tests and packaging tests updated for 1.0 layout.
- Migration guide: [MIGRATION-0.7-to-1.0.md](../MIGRATION-0.7-to-1.0.md).
- **541** engine tests pass; Studio wheel smoke test passes on `campaign_rpg_engine-1.0.0` wheel.

---

## Release

1. Tag **`v1.0.0`** and push to GitHub.
2. `uv build` → `uv publish` to PyPI.
3. Bump [CampAIgn-RPG-Studio](https://github.com/DaveH-Ghost/CampAIgn-RPG-Studio) to PyPI-only `campaign-rpg-engine>=1.0.0` (remove `[tool.uv.sources]` path override).

Save format **`snapshot_version`** remains **4** — no bump required for 1.0.
