# Version 1.3.0 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Scene decorations — visual-only map layers (background + sprites) stored per area, excluded from LLM prompts and turns.

**Baseline:** **1.2.1**

---

## Additions

- `Decoration` model with `kind: "sprite" | "background"`, `image`, `z_index`, and sprite geometry (`x`, `y`, `width`, `height`) or background `repeat`.
- `Area.decorations` list with lookup/remove helpers.
- Session typed API: `create_decoration`, `update_decoration`, `delete_decoration`, `reorder_decoration`.
- `DecorationMutationResult` for decoration CRUD outcomes.
- `serialize_decoration` / snapshot `decorations[]` on each area block.

## Persistence

- **`snapshot_version` 5** — each area block may include `decorations: []`.
- v4 saves load with empty decorations (backward compatible).

Decorations are **not** included in prompt builders, passive vision, or turn simulation.
