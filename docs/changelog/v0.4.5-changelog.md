## Save format (`snapshot_version: 1`)

Distinct from `Session.snapshot()` (API view for realm-studio `/api/state`).

Top-level fields: `snapshot_version`, `engine_version`, `profile_id`, `include_examples`, `session_turn`, `active_agent_id`, `active_area_id`, `vision_units`, `vision_units_per_tile`, `prompt_blocks` (null = profile default), `agent_area`, `areas`, `agents`.

Per-agent `memory` block:

- `looked_at`, `ever_looked` — passive vision knowledge (Memory facade)
- `module_id`, `module_state` — from `MemoryModule.export_state()` (config + runtime buffers, not prompt text)

Rolling-summary consolidation thread state is **not** persisted; load always starts idle.

## Engine API

```python
from realm_fabric import Session, build_save_snapshot, load_session_from_snapshot

data = session.to_save_dict()          # or build_save_snapshot(session)
restored = Session.from_snapshot(data)  # or load_session_from_snapshot(data)
```

Pre-save: `ensure_ready_for_turn()` on each agent so rolling-summary consolidation finishes.
