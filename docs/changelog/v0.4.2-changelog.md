## 0.4.2a — Action range in session units

**Status:** ✅ **Done**

Object interaction lines in the prompt use `format_action_range_label()` when session **Units** and/or **Units per tile** are set:

- Default: `range 1` (Chebyshev tiles)
- With `ft` and 5 per tile: `range 5 ft` for range 1; `range 10 ft` for range 2
- `range 0` stays `same tile`

Threaded through `look_and_interact` slot rendering (with vision units).

## 0.4.2b — Speak as optional step

**Status:** ✅ **Done**

**Before:** `turn_action` was `speak | interact | none` — speak and interact were mutually exclusive.

**After:** Pipeline is **move → look → speak → turn action (interact | emote | none)**:

- `content` — optional speak dialogue (independent of `turn_action`)
- `turn_action` — `interact`, `emote`, or `none`
- Agents may speak **and** interact (or emote) in the same turn

Updated: `AgentCompoundTurn`, simulation, compound rules, output format, few-shots, `step-compound` parser.

**Breaking:** JSON with `"turn_action": "speak"` is rejected; use `"turn_action": "none"` with `content` set.

## 0.4.2d — Emote turn action

**Status:** ✅ **Done**

**Pipeline:** move → look → speak → turn action (`interact` | `emote` | `none`).

| Field | Emote |
|-------|--------|
| `turn_action` | `"emote"` |
| `target` | Entity id or free text |
| `action_name` | Past-tense verb from LLM (e.g. `pointed`, `smiled`) |

**Result / passive phrasing:**
- Object target → `the {name}` (e.g. `You pointed at the Wooden Sign.`)
- Agent target → `{name}` (e.g. `You smiled at Goblin.`)
- Self-target → `yourself` for actor result
- Witness who **is** the target → `{actor} {verb} at you.`

No visibility or range checks in v1. Prompt rules in `look_and_interact` slot include past-tense guidance and examples.

**Removed (0.4.2):** `confidence` and `emotion` JSON fields — use `turn_action: "emote"` for non-verbal expression instead.
