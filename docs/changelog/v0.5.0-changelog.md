## Editor UX (scroll + collapse)

- Lorebooks tab fills viewport height; entry list scrolls inside the editor panel
- Loaded-books list scrolls when many books are present
- Entries default to **collapsed** (label + enable/constant only); expand per entry or use **Expand all** / **Collapse all**
- **Save changes** stays pinned at the bottom of the editor

## Add and delete entries

- **Add entry** creates a blank session entry (new `uid` / `order`) in the editor
- **Delete** removes an entry from the session copy (with confirmation)
- **Label** field edits the collapsed header text
- Status line shows unsaved vs saved entry count

## Download lorebook (ST export)

- **Download** on each book in the list; **Download JSON** in the editor footer
- `GET /api/lorebooks/{id}/download` â€” SillyTavern load shape (`entries` map keyed by uid string)
- Filename from original upload or `{id}.lorebook.json`
- Includes fields Realm Fabric does not use at match time so exports work in SillyTavern and similar tools

## SillyTavern field compatibility

- Import uses **`disable` only** (not `enabled`) to match ST lorebooks
- realm-studio **Enabled** checkbox is a readable inverse of `disable`; export writes `disable`
- Unused ST fields are stored per entry in `raw` and preserved across studio **Save** (merged by `uid`)
- Export / download fills any **missing** ST keys from defaults (see below) without overwriting values already on the entry

## ST entry defaults (`src/lorebook/st_defaults.py`)

New session entries and export normalization include typical ST fields from a standard export (e.g. `tri-system-universe.lorebook.json`):

| Field | Default |
|-------|---------|

Realm Fabricâ€™s matcher still ignores these unless noted elsewhere (e.g. `caseSensitive` in `raw`); they exist for round-trip fidelity.

## Keyword scan sources (configurable)

- Session setting `lorebook_scan_config` â€” enable/disable each source used for keyword matching
- Sources: agent name, personality, description; area description; memory module text; **passive vision**; recent area events
- Matching is per **active agent** at prompt build time (constants still always inject when enabled)
- realm-studio **Lorebooks** tab: right-hand **Keyword scan sources** panel with checkboxes and live previews for the active agent
- API: `GET/PUT /api/lorebooks/scan-config`
- Saved in session snapshot (`lorebook_scan_config`)
