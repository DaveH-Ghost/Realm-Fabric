# Version 1.3.1 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Area templates — save and load a whole area as a reusable blueprint (bounds, decorations, objects, agents with positions).

**Baseline:** **1.3.0**

---

## Additions

- `export_area_template(session, area_id, *, name, include_hidden_objects)` — portable area document (`kind: "area"`); objects and decorations only (no agents).
- `export_decoration_template(decoration)` — decoration entry without id.
- `spawn_area_from_template(session, template, *, area_id, mode="new"|"replace")` — create or replace an area from a template.
- `validate_area_template(data)` — schema validation for area template JSON.
- `AreaTemplateMutationResult` for spawn outcomes.

Area templates compose entity template export/spawn internally. Object ids are regenerated on load (session-wide uniqueness); positions are preserved. Agents are not included in area templates.

## Fixes

- Auto-generated object ids (`obj_*`) are unique across **all areas** in a session, not per-area only.
