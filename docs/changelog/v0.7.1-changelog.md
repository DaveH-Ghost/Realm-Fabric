### PyPI `__version__` read fix

`campaign_rpg_engine.__version__` now reads from installed package metadata (`importlib.metadata`) when available, with `pyproject.toml` fallback for editable/path installs. Fixes incorrect version reporting when installed from PyPI (needed for apps that pin `campaign-rpg-engine>=0.7.1`).

## Changes (integration)

### Straight-line movement drift (move_speed)

When `move_speed` is limited and the agent moves along a straight horizontal or vertical line (e.g. `(0, 2)` → `(4, 2)` with speed 3), BFS pathfinding treated diagonal zigzags as equally short paths and the agent could stop one tile **off** the line — e.g. `(3, 3)` or `(3, 1)` instead of `(3, 2)`.

**Fix:** `find_path` tie-breaks collinear start/goal pairs to prefer staying on the shared row or column. Diagonal-first tie-breaking still applies when both axes differ (5e-style).

**Tests:** `test_move_speed_straight_horizontal_stays_on_row`, `test_move_speed_straight_vertical_stays_on_column` in `tests/test_move_pathing.py`.

**Status:** ✅ Done

### Targeted area events

`Session.emit_area_event(text, agent_ids=...)` can deliver narrator/GM events to specific agents.

| `agent_ids` | Behaviour |
|-------------|-----------|
| Omitted or empty | All agents in the **active area** receive the event; appended to `recent_events` (unchanged 0.7.0 behaviour) |
| One or more ids/names | Only those agents receive the event in memory; **not** added to `recent_events` |

Agents are resolved by id (`agent_*`) or display name across the session (including multi-area).

**campaign-rpg-studio:** **Emit event…** includes an optional multi-select recipient list; leave empty to broadcast to the active area.

**Tests:** `test_emit_area_event_targeted_agent`, `test_emit_area_event_targeted_by_id`, `test_emit_area_event_unknown_agent_fails`, `test_emit_area_event_empty_agent_ids_broadcasts_all` in `tests/test_area_events.py`; `test_post_event_targeted_agents` in campaign-rpg-studio API tests.

**Status:** ✅ Done
