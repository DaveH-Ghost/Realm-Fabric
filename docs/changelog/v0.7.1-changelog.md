## How implementation works (separate repo)

1. Create the demo repository (e.g. `realm-fabric-sample`) with its own `pyproject.toml`, `README`, and changelog starting at **`0.1.0`**.
2. During co-development, depend on Realm-Fabric via **path**, **git tag**, or **PyPI**:
   ```toml
   [tool.uv.sources]
   realm-fabric = { path = "../Realm-Fabric", editable = true }
   ```
3. Build the playable loop in the **demo repo** (handlers, UI, scenario, HTTP routes).
4. When the demo hits a **missing export**, **typed API gap**, or **engine bug** → implement and test **in this repo**, document in **this changelog**, tag **`v0.7.1`** (or `0.7.2`, …).
5. Bump the demo’s `realm-fabric` requirement and continue on demo **`0.1.x`** / **`0.2.0`** as appropriate.
