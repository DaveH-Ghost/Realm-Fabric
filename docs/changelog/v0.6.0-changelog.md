## Save format (`snapshot_version: 3`)

New saves written by **0.6.0** use **`snapshot_version: 3`**. Import of **v1** and **v2** saves is still supported (missing fields get safe defaults).

### Additive object fields (since 0.6.0a / 0.6.0d / 0.6.0e)

| Field | Default | Notes |
|-------|---------|--------|
| `blocks_movement` | `true` (objects) | All footprint tiles when blocking |
| `movement_exceptions` | `[]` | Entity ids that may pass / stand |
| `width` / `height` | `1` | Footprint anchor at `position` |
| `hidden` | `false` | Excluded from agent vision when true |
| `actions_detail[].kind` | `"interact"` | `"trigger"` for path-step triggers |
| `actions_detail[].halt_movement` | `false` | Trigger-only |
| `actions_detail[].delete_after_trigger` | `true` | Trigger-only |
| `actions_detail[].trigger_exceptions` | `[]` | Agent ids that skip trigger |

### Additive agent fields

| Field | Default | Notes |
|-------|---------|--------|
| `blocks_movement` | `false` | Agents non-blocking by default |
| `movement_exceptions` | `[]` | Same semantics as objects |
| `is_player` | `false` | Manual compound turns in studio |
| `private_data` | `""` | App-owned text; not used by engine or LLM |

### `private_data` (0.6.0f)

- On agents and objects in snapshots/saves and `GET /api/state`
- **Not** in CLI, LLM prompts, or simulation
- realm-studio edits via `PUT /api/entity-private-data` (Advanced group)

## 0.6.0a — Movement blocking + pathfinder ✅

### Engine

- **`src/occupancy.py`** — tile enterability, `movement_exceptions`, `resolve_standable_goal` (nearest standable tile when goal is blocked)
- **`src/pathfinding.py`** — BFS pathfinding (8-way, cost 1 per step, same 5e rules as before); `walk_with_pathfinding` returns final position, reached flag, and path steps
- **`Object.blocks_movement`** — default **`true`** (objects block their tile)
- **`Object.movement_exceptions`** — entity ids allowed to pass through / stand on that object’s tile
- **`Agent.blocks_movement`** — default **`false`** (agents can share tiles, D&D-friendly)
- **`Agent.movement_exceptions`** — same exception list semantics for blocking agents
- **`move()`** — limited `move_speed` uses pathfinder; unlimited move walks path tiles for triggers and evaluates enterability
- Move to **entity id** resolves to the object/agent tile, then paths to nearest **standable** tile when that tile blocks movement

### CLI

```
create-object ... [blocks-movement true|false] [movement-exception id1,id2]
edit-object <id> ... blocks-movement false movement-exception agent_01
create-agent ... [blocks-movement true|false] [movement-exception ...]
edit-agent <id> ... blocks-movement true movement-exception obj_door_01
```

### Save / snapshot

- Object and agent JSON include `blocks_movement` and `movement_exceptions`
- **0.6.0a–d** stayed on **`snapshot_version: 2`** (additive only); **0.6.0e** bumps to **3**

### Tests

- `tests/test_movement_blocking.py`
- `tests/test_move_pathing.py`, `tests/test_move_target.py`, `tests/test_session.py` updated for blocked tiles

### realm-studio (0.6.0a)

- **Create object** / **Edit object** — **Blocks movement** checkbox (default on); **Movement exceptions** comma-separated entity ids when blocking is enabled
- Removed **Add eat action** checkbox from create object (use **Manage actions…** instead)

### Breaking / behavior changes

- Agents **no longer teleport onto** blocking object tiles when moving to `obj_*` targets; they stop on the nearest standable adjacent tile
- Greedy `walk_towards` in `src/pathing.py` remains for unit tests; **runtime move** uses BFS when `move_speed` is set
- Cookie-style demos that need same-tile interact (`range 0`) should use **`blocks-movement false`** on the object

### Move result clarity

- **Coordinate partial moves** — first-person result includes stop tile: `You moved towards (4, 4), stopping at (3, 2).`
- **Entity partial moves** — no coordinates; distance only: `You moved towards Ceramic Ball; you are still 2 steps away.`
- **Blocking entity arrival** — adjacent stop counts as success: `You have successfully moved next to Ceramic Ball.`
- **Already adjacent** — `You are already next to Ceramic Ball.` (not the object’s tile coordinate)
- **Blocked coordinate** — `You are already at (1, 1), as close as you can get to (2, 2).`
- **Unreachable entity** — names the blocker: `You cannot reach Ceramic Ball; Wall is blocking the way.`
- **`src/occupancy.py`** — `find_blocker_between()` for blocker identification on failed paths

### Player agents (realm-studio testing)

- **`Agent.is_player`** — human-controlled agent; same simulation rules, no LLM on turn
- **CLI:** `create-agent ... player true`, `edit-agent <id> player true|false`
- **Snapshot / save:** `is_player` on agent JSON (default `false`)
- **realm-studio:** **Player** checkbox on create/edit agent; **Run turn ▶** opens manual compound-turn form instead of calling the LLM
- **`POST /api/turn/manual`** — validate and run `AgentCompoundTurn` JSON for player agents

## 0.6.0b — Interact pathing (overwrites move) ✅

### Engine

- **`interact()`** — if out of `action.range`, spends **`move_speed`** pathing toward the nearest enterable tile from which the action is legal, then executes if in range
- Unlimited **`move_speed`** (`None`) walks the path tile-by-tile (including trigger evaluation) when pathing for interact
- **`execute_nav_phase`** — when compound turn `action` is **`interact`**, the explicit **`move`** field is **ignored** (no separate nav step); pathing happens inside interact

### Compound turn semantics

- Order unchanged in record: **move step omitted** → look → speak → interact/emote
- LLM may still send `move` + `interact` in one JSON turn; engine uses interact-path only
- **Interact pathing** records a **move** turn step (move-style result/passive) when pathing changes the agent's tile, so memory includes where they stopped

### Tests

- `tests/test_interact_pathing.py`
- `tests/test_object_actions.py` updated (far cookie + `move_speed 1` for out-of-range cases)

### Notes for smoke testing

- Kick ball from distance: one compound turn with `action: interact` should path adjacent (if `move_speed` allows) and kick without a separate move
- `step-compound 2,3 interact obj_… eat` — move token is ignored when interact is present; agent paths via interact rules only

## 0.6.0e — Hidden objects + triggers ✅

### Engine

- **`Object.hidden`** — excluded from passive vision and look; still in saves and GM tooling
- **`edit-object … hidden true|false`** — hide or reveal objects
- **`ObjectAction.kind`**: `interact` | `trigger`
- Trigger fields: **`halt_movement`**, **`delete_after_trigger`** (default true), **`trigger_exceptions`**
- **`src/triggers.py`** — fires on each path step during move and interact pathing; once per turn per `(agent, object, action)`; outcome via **`Session.emit_area_event()`** from `passive_result`
- **Multi-tile triggers** — use object **`width` / `height`** footprint (tripwires, room-entry cutscene zones); range is Chebyshev to the **nearest footprint tile**
- **`snapshot_version: 3`**
- **Deferred to 0.6.1** — trigger **effects** / handlers (same rework as interact effects; area events only for now)

### CLI

```
create-object … hidden true width 3 height 1 blocks-movement false
edit-object <id> hidden false
edit-object <id> add-action trip range 0 kind trigger halt-movement true \
  delete-after-trigger false result "(trigger)" passive "{actor} steps on the trap."
```

Multi-tile tripwires and room-entry zones: set **`width` / `height`** on the hidden object; **`range 0`** fires when the agent steps on any footprint tile.

### realm-studio

- Hidden checkbox on create/edit object; ghost overlay markers for hidden objects
- **Create hidden trigger…** on empty tile context menu (object + trigger in one flow); **width / height** for multi-tile tripwires and room-entry zones
- **Manage actions…** — kind `interact` | `trigger` with trigger-specific fields
- Object context menu **Hide** / **Reveal**
- **`POST /api/command`** returns updated **snapshot** on success

### Tests

- `tests/test_hidden_objects.py`, `tests/test_triggers.py`
- `examples/web/realm-studio/tests/test_api.py` — hidden + trigger commands
