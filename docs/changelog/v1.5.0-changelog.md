# Version 1.5.0 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Affinity memory module — rolling chronicle plus per-agent relationship scores (-10…+10) with window-only consolidation.

**Baseline:** **1.4.2**

**Status:** Implemented — tag **`v1.5.0`**.

---

## Additions

### Built-in memory module ``affinity``

Extends rolling-summary buffers with:

- Per other-agent **affinity** (-10…+10) + short relationship summary
- Prompt block order: **Relationships** → **Summary** → detail turns
- Consolidation every ``summary_interval`` turns runs **Call A** (overall summary) and **Call B** (affinity deltas) **in parallel**
- Call A sees only previous summary + turn batch (no affinity state)
- Call B sees only turn batch + current affinity rows / ±1 guidance (no rolling summary)
- All-or-nothing apply; owner gated until both succeed; others may act; live post-snapshot witnesses retained
- Same create-agent flags as rolling summary: ``memory-summary-interval``, ``memory-summary-max``, ``memory-summary-tail``

### Removed: custom memory modules

- ``register_memory_module_from_path`` / ``register_memory_module_from_source`` and the loader are **gone**.
- Built-ins only: ``recent_turns``, ``salient_turns``, ``rolling_summary``, ``affinity``.
- Snapshot / template import of unknown ``module_id`` fails with ``unknown_memory_module_message``.

### Supporting APIs

- ``AffinityModule``, affinity ladder helpers, ``generate_affinity_updates``
- ``ConsolidationSnapshot.extra`` for module-specific freeze data
- ``Memory.record_turn(..., nearby_agents=...)`` for same-area candidate detection
- ``MemoryRecordContext.nearby_agents``

---

## Compatibility

- **``snapshot_version``** remains **5**.
- Existing ``rolling_summary`` unchanged.
- Saves that use ``affinity`` require this engine version (and module id registered).
