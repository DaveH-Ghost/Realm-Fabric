## Changes

### Python 3.12 import fix (`from __future__ import annotations`)

PyPI wheels could raise `NameError` on Python 3.11/3.12 when importing `campaign_rpg_engine` (e.g. forward references like `-> GridBounds` inside `class GridBounds`, or types only imported under `TYPE_CHECKING`). Python 3.14 was more lenient in local dev, which hid the issue until server installs.

**Fix:** Add `from __future__ import annotations` to every `src/**/*.py` module that was missing it (~30 files). Defers annotation evaluation so forward references resolve correctly.

**Tests:** `test_src_modules_have_future_annotations`, `test_campaign_rpg_engine_imports_from_built_wheel` in `tests/test_packaging.py`. Verify on Python 3.12 before publish:

```powershell
uv run --python 3.12 pytest tests/test_packaging.py -q
```
