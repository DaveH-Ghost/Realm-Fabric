# Version 1.7.3 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Joint release with Studio **1.7.3** — turn-verb pathing parity with interact, dynamic verb ranges, and an `area_event` session hook for plugins (combat start/end tags).

**Status:** Implemented — tag **`v1.7.3`** on publish.

## Changes

### Turn verb pathing

- Honor an explicit compound `move` when it already places the actor in the verb’s Chebyshev range of the path target (same idea as interact).
- `register_turn_verb(..., path_range_from_turn=...)` — optional dynamic range resolver `(session, agent, area, turn) -> int | None`; overrides static `path_range` when it returns an int.
- `explicit_move_reaches_agent_range`, `resolve_verb_path_range` helpers.

### Area events

- `Session.emit_area_event` fires session event `area_event` with `text`, `broadcast`, and `agent_ids` so Studio plugins can react (e.g. `[Combat Start]` / `[Combat End]`).

## Compatibility

- CampAIgn-RPG-Studio **1.7.3** requires **`campaign-rpg-engine>=1.7.3`**.
