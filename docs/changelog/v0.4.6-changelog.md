## Custom memory modules

- **`_BUILTIN_REGISTRY`** — always loaded (`recent_turns`, `salient_turns`, `rolling_summary`)
- **`_CUSTOM_REGISTRY`** — runtime only; register via path or uploaded source
- **Overwrite** — re-registering the same `MODULE_ID` replaces factory + cached file
- **Reject** — `MODULE_ID` cannot match a built-in id

```python
MODULE_ID = "my_module"          # required
MODULE_LABEL = "..."             # optional catalog
MODULE_DESCRIPTION = "..."       # optional
CREATE_AGENT_OPTIONS = [...]     # optional create-agent flags
def create_module(**config) -> MemoryModule: ...
```

Example: [examples/custom_memory/rolling_summary_custom.py](../../examples/custom_memory/rolling_summary_custom.py).

## Save/load validation

`validate_snapshot_modules(data)` runs before session mutation on import. If any agent's `memory.module_id` is not loaded:

```
Memory module '{module_id}' is not found. Load the module before loading this save.
```

Saves still use `snapshot_version: 1` with `module_id` + `module_state` only — **no** `module_source` in JSON.

## campaign-rpg-studio

- **Settings** (gear, next to save/load): API key (password; shows configured, never returned), model text, **Add module** `.py` upload, list of loaded custom modules
- **`GET/PUT /api/settings/llm`** — in-memory env only
- **`POST /api/memory-modules/upload`** — multipart `.py` → cache + register
- **`GET /api/memory-modules`** — only **loaded** modules (built-ins + customs)
- Create-agent memory dropdown: no “Custom” option — upload first, then pick from catalog

Windows SAC workaround: `uv run python -m backend.main` instead of `uv run campaign-rpg-studio`.
