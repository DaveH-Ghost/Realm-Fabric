# Version 1.6.0 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Professional hardening — lint/CI, package layout (`edit/`, `templates/`), Mermaid UML, Studio positioned as GM host.

**Baseline:** **1.5.2**

**Status:** Implemented — tag **`v1.6.0`** on publish.

---

## Additions

### Tooling

- **Ruff** lint + format (`uv run ruff check` / `ruff format`).
- GitHub Actions CI: ruff + pytest.

### Package layout

- `campaign_rpg_engine/edit/` — `area_edit`, `area_edit_parse`, `world_edit_api`, `session_area_edit`, `decoration_edit`, plus `field_appliers` (shared field application; untangles circular imports).
- `campaign_rpg_engine/templates/` — `entity_templates`, `area_templates`, `interact_templates`.
- **Compat shims** at old top-level module paths (`campaign_rpg_engine.area_edit`, etc.) so Studio and plugins keep working.
- Public `Session` / `__all__` unchanged.

### Docs

- [`docs/UML/`](../UML/README.md) — Mermaid diagrams (system overview, turn flow, package layout, edit stack) + [structural audit](../UML/AUDIT-1.6.0.md).
- Product language: CampAIgn-RPG-Studio is the **GM host**, not a “reference demo.” Player clients attaching later are documented as future intent.

---

## Compatibility

- Import paths under `campaign_rpg_engine.edit.*` / `templates.*` are preferred for new code; shims remain for 1.6.x.
- CampAIgn-RPG-Studio **1.6.0** requires **`campaign-rpg-engine>=1.6.0`**.
