# Version 1.7.2 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Joint release with Studio **1.7.2** — Featherless / one-at-a-time LLM hardening, affinity resilience, and consolidation UX for hosts.

**Baseline:** **1.7.1**

**Status:** Implemented — tag **`v1.7.2`** on publish.

---

## Changes

### Concurrent LLM policy

- Process flag ``LLM_CONCURRENT_CALLS`` / ``set_concurrent_llm_calls`` / ``concurrent_llm_calls_enabled``.
- When concurrent calls are **disabled**: rolling-summary and affinity consolidations run **synchronously** before the turn returns; affinity Call A then Call B run **sequentially** (needed for Featherless Premium + DeepSeek’s 4-unit concurrency).
- ``ConcurrencyLimitError`` / ``is_concurrency_limit_error`` classify provider concurrency rejections (429 + concurrency signals; not brittle full-string match).
- Failed consolidations caused by concurrency limits set ``MemoryConsolidationError.concurrency_limit_exceeded`` so Studio can offer recovery UI.
- ``notify_consolidation_failure`` / ``register_consolidation_failure_listener`` so hosts can surface consolidation failures immediately (SSE), not only on the next gated turn.
- Saves/checkpoints use ``flush_for_save()`` (wait only; do not raise) so one agent’s failed consolidation cannot crash another agent’s turn path.

### Affinity / rolling-summary prompts & parsing

- Consolidation prompts include an identity preamble: personality, passive description (“appearance”), and other agents in the area (name + passive description) so pronouns and relationships stay grounded.
- Affinity Call B JSON repairs for Featherless bracket quirks: missing ``[``, trailing ``}`` instead of ``]``, and outer curly wrapper ``{{...},{...}}`` → ``[{...},{...}]``.
- Affinity Call B never treats GM area broadcasts (``__area__`` / Environment) as relationship candidates.
- Consolidation / affinity failure logs print **Prompt** and **Raw Model Output** above the ERROR panel when available.
