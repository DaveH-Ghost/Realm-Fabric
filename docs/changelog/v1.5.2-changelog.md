# Version 1.5.2 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Multi-provider LLM client (OpenRouter + Featherless) and hard max input-token budget.

**Baseline:** **1.5.1**

**Status:** Implemented — tag **`v1.5.2`** on publish.

---

## Additions

### LLM providers

- `LLM_PROVIDER` selects `openrouter` (default) or `featherless`.
- Each provider uses its own API key / model env vars (`OPENROUTER_*` / `FEATHERLESS_*`) and base URL.
- DeepSeek V4 Flash ids differ by catalog: OpenRouter `deepseek/deepseek-v4-flash`, Featherless `deepseek-ai/DeepSeek-V4-Flash`.
- Existing OpenRouter-only setups keep working with no new required vars.

### Input token budget

- Default max estimated input tokens: **32768** (`LLM_MAX_INPUT_TOKENS`).
- Soft warning percent: **90** (`LLM_INPUT_WARNING_PERCENT`) — exposed for hosts; does not refuse calls.
- `prompt_token_budget_status` / `prompt_exceeds_max_input` helpers.
- `get_structured_turn`, `get_compound_turn`, and `get_text_completion` raise `PromptTooLargeError` before the HTTP call when the estimate exceeds the max.

### Env example

- `.env.example` documents both providers plus budget vars (DeepSeek V4 Flash ids differ by catalog).

### JSON parse repair

- If structured LLM output fails to parse and looks like a JSON object **missing the leading `{`** (seen with Featherless DeepSeek V4 Flash), prepend `{` and retry once before raising `LLMParseError`.
- `LLMParseError` carries `raw_response` for hosts to surface failed model text.
