# Version 1.5.1 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Soak-test interaction and prompt UX — enabled actions, `[far]` / `[emote]` tags, clearer interact failures, approach pathing.

**Baseline:** **1.5.0**

**Status:** Implemented — tag **`v1.5.1`** on publish.

---

## Additions

### `ObjectAction.enabled`

- Actions may be disabled (`enabled: false`); they are hidden from passive vision and cannot be used or fire as triggers.
- CLI / create-object / add-action support `enabled true|false` (default true).
- Round-trips in snapshot serialization.

### Passiveive vision — `[far]` actions

- Out-of-range object actions are still listed, tagged `[far]`, with a top-of-vision rule.
- Prefer `move` toward the object (speak/emote on the way); `interact` still walks closer but may not finish this turn.

### Emotes — third-person `[emote]` tag

- Actor result and passive lines use third person (no leading `You …`).
- Lines are prefixed `[emote]`; compound rules explain witnesses remember them and they can matter socially without moving entities or changing objects.

### Interact verb validation

- Unknown interact verbs fail with available action names (or “has no interact actions”) and a nudge to use `emote` for unlisted roleplay.
- Compound rules require exact verb names from passive vision; do not invent interact verbs.

### Handlers / CLI

- Extra field keys may start with a digit (e.g. `1_set_pdesc`) so sequence nested params parse correctly.

### Approach pathing

- `resolve_standable_goal(..., from_pos=)` prefers nearest-to-mover standable tiles (diagonal approach fix).

### Optional emote target

- Emote may omit target (undirected beat).

---

## Compatibility

- **`snapshot_version`** remains **5**.
- Saves with `enabled` on actions load on older engines as always-enabled if the field is ignored; prefer Studio/engine **1.5.1+** together.
