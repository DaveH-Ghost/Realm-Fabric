## Why V0.4.1

V0.4.0 made worlds bigger and movement smarter. The next pain point in live play is **turn reliability** and **prompt iteration**:

1. **Failed turns from length limits** — `AgentCompoundTurn` currently **rejects** JSON when `reasoning` exceeds 400 characters or speak `content` exceeds 500 characters (or more than five sentences). Models do not count reliably while planning; hard validation wastes LLM calls.
2. **Prompt editing is file-only** — `GameProfile` loads `profiles/default_compound/template.txt` with `{{slot}}` placeholders. GMs using realm-studio cannot reorder sections or edit house rules without leaving the browser or redeploying files.

V0.4.1 keeps the same compound turn schema and execution order; it changes **how we handle overlong fields** and **how prompts are assembled** for a session.

**Deferred:** swappable turn schemas / alternate Pydantic executors → **V0.5+**; profile export to disk; editing engine-computed slot *content* (passive vision, memory) as if it were prose.

## Truncation — design (0.4.1a)

### Today

- `src/llm/schemas.py` — `_validate_reasoning` raises `ERR:REASONING_TOO_LONG` above 400 chars.
- `_validate_speak_content` raises above **500 characters** or **5 sentences**.
- Prompt rules and `compound_output_format()` tell the model these hard caps.
- Failed validation → turn does not run; user sees parse/validation error.

### Target

**Truncate at parse time; never fail a turn solely for length.**

| Field | Budget | Rule |
|-------|--------|------|
| `reasoning` | 400 characters | Sentence-aware truncation (see below) |
| `content` (speak) | 500 characters | Same |

**Sentence-aware algorithm (shared helper):**

1. Normalize ellipsis runs (`...` → single character) so they are **not** sentence boundaries (same idea as today’s `count_speak_sentences`).
2. Split into sentences; record each sentence’s **start index** in the original string.
3. **Keep** every sentence whose start index is **strictly before** the budget index (0-based: `start < max_chars`).
4. **Drop** all sentences that start at or after the budget.
5. If a kept sentence extends past the budget, keep it **whole** — no mid-sentence cuts.

**Examples (speak, budget 500, 1-based char positions for intuition):**

| Case | Result |
|------|--------|
| Sentence A ends at 480; sentence B starts at 481 and runs to 620 | Keep A + **full B** |
| Sentence A ends at 480; sentence B starts at 501 | Keep A only |
| Single run-on sentence starting at 1, length 800 | Keep entire sentence (started before budget) |

**Drop the five-sentence speak cap** — character budget is enough and maps more cleanly to token planning.

**Prompt copy:** Soft guidance (“aim for ~400 / ~500”); note that longer text is trimmed at sentence boundaries server-side. Remove “max 400” / “five sentences” as hard requirements in rules and JSON examples.

**Optional later:** log or surface when truncation occurred (realm-studio turn log / debug).

### Out (0.4.1a)

- Truncating `confidence` / `emotion` (still short optional fields).
- Token-based limits (chars remain the v1 contract).

## Prompt blocks — design (0.4.1b–c)

### Today

- `build_prompt_context()` → `PromptContext` (engine-built strings each turn).
- `prompt_context_slots()` maps slots: `character`, `passive_vision`, `memory`, `area_description`, `grid_description`, `compound_rules`, `move_instructions`, `look_and_interact`, `output_format` (+ alias `rules` → `compound_rules`).
- `GameProfile.template` — single `template.txt` with `{{slot}}` substitution; **order = file order**.
- `Session.profile` is fixed at construction; realm-studio only **reads** prompt via `GET /api/prompt`.

### Target: ordered block list (Option B)

Session holds an ordered list of **blocks** rendered into the final prompt. Default block list is derived from `profiles/default_compound/template.txt` (one-time migration / loader).

**Block types:**

| Type | Purpose | GM editable |
|------|---------|-------------|
| **`slot`** | Insert engine value: `passive_vision`, `memory`, `move_instructions`, … | Reorder; optional wrapper via adjacent `text` blocks |
| **`text`** | Literal glue: `"Passive Vision:\n"`, blank lines | Full text |
| **`section`** | Static prose stored on the session: `compound_rules`, `output_format` | Full body (replaces hardcoded strings from `prompt_context.py` for those sections when overridden) |

**Dynamic vs static:**

- **Dynamic slots** — recomputed each turn from sim state. GM edits **layout** (order, labels), not the live vision/memory text.
- **Section blocks** — GM-editable house rules and JSON shape instructions.

**Rendering:** walk blocks in order → resolve slots from `PromptContext` + session section overrides → concatenate.

**Persistence v1:** in-memory on `Session` (realm-studio process lifetime). Reset restores profile default. Export/import JSON and writing back to `profiles/` → later.

### API (realm-studio, 0.4.1b)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/prompt-slots` | Slot catalog: name, description, current rendered preview for active agent |
| `GET` | `/api/prompt-blocks` | Session block list (or default) |
| `PUT` | `/api/prompt-blocks` | Replace block list |
| `GET` | `/api/prompt` | Unchanged — full rendered prompt (uses blocks) |

### realm-studio UI (0.4.1c)

- New sidebar panel or modal: **Prompt layout**

## 0.4.1c+ — Slot settings, vision units, and prompt polish

**Status:** ✅ **Done**

**Goal:** Let GMs tune *how* dynamic slots render (without editing live sim text) and add session-scoped distance units for spatial prompts.

### Session vision units

| In | Detail |
|----|--------|
| **`Session.vision_units`** | Optional unit label (letters only, e.g. `ft`) |
| **`Session.vision_units_per_tile`** | Optional positive integer multiplier per grid tile |
| **`Session.set_vision_units()`** | Validation + update; included in snapshot |
| **API** | `PUT /api/vision-units` — `{ units, units_per_tile }` |
| **realm-studio** | **Units** / **Units per tile** fields under Session meta (`visionUnits.js`); auto-save on blur |
| **Prompt render** | Units threaded through `GameProfile.build_prompt` → `render_prompt_blocks` for slots that need them |

When **Units per tile** is unset, distance math defaults to **1 unit per tile**. When **Units** is unset, move speed falls back to `step(s)`.

### Slot ⚙ settings (`SLOT_SETTINGS` in `prompt_blocks.py`)

Per-slot options stored on `PromptBlock.options`; catalog exposes field defs via `GET /api/prompt-block-catalog` → `slot_settings`. realm-studio **Prompt layout** panel shows ⚙ on configured slots.

| Slot | Toggles | Notes |
|------|---------|-------|
| **Character** | Name, Personality, Description | At least one must stay enabled |
| **Passive vision** | You are at, Entity coordinates, Direction and distance | Bearing requires session units + units per tile |
| **Move instructions** | Coordinate moves | Omits `format_move_coordinate_rule()` paragraph when off |

Slots without ⚙ (memory, area/grid description, look_and_interact, …) render as before.

### Passive vision — direction and distance

| In | Detail |
|----|--------|
| **`src/vision_bearing.py`** | Eight-zone compass from tile offset (screen coords: Y increases **south**) |
| **Phrase** | `{direction} of you, {distance} {units} away`; same tile → `on your tile` |
| **Distance** | Chebyshev tile distance × `units_per_tile` |
| **Entity lines** | Bearing applies to objects and other agents; `(x, y)` on entity lines is independent of bearing toggle |
| **Tests** | `tests/test_vision_bearing.py`, `tests/test_perception.py`, prompt-block slot tests |

Example entity line with all toggles on and `ft` / `5` per tile:

```
Ceramic Ball (obj_ball_01), (2, 2), South-East of you, 10 ft away - [?]
```

### Move instructions refactor

| In | Detail |
|----|--------|
| **Entity targets** | Single line: set `move_target` to `obj_*` / `agent_*` (removed per-entity target list) |
| **Coordinate moves** | Optional paragraph from `area.format_move_coordinate_rule()` |
| **Move speed line** | `move_speed × units_per_tile` with session unit label; e.g. speed 2, 5 ft/tile → `Your move speed this turn is 10 ft.` |
| **Removed copy** | Diagonal/straight cost note and “stop short of target” (engine still pathfinds; not prompt clutter) |
| **Tests** | `tests/test_move_target.py`, `tests/test_move_pathing.py`, prompt-block tests |

### Grid description fix

| In | Detail |
|----|--------|
| **`area.format_grid_description()`** | Northwest → southeast bounds; **Y increases southward** (matches realm-studio grid) |

### Out (0.4.1c+)

- ⚙ on remaining slots (memory, look_and_interact, output_format, …) — **0.4.2+** / engine work as needed
- Editing passive vision *content* (object descriptions) through the block editor
- Token/image format work (PNG already works via `<img>`; no engine change)
