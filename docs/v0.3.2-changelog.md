# Version 0.3.2 Changelog

**Purpose:**  
Living record of what ships in **V0.3.2** and why — **not** a pre-implementation checklist. Each section corresponds to an incremental release slice (e.g. `0.3.2a`, `0.3.2b`) so we can implement and review agilely without a full spec up front.

**Baseline:** Builds on **V0.3.1** (`v0.3.1`) — [realm-studio](../examples/web/realm-studio) with grid UI, right-click editing, LLM **Run turn**, passive vision, and turn log. Engine remains **`0.3.0`** in root `pyproject.toml` unless a slice needs a small **`0.3.0.x`** patch with tests.

## Why V0.3.2 (not multiplayer)

V0.3.1 proved the engine works through HTTP. **0.3.2** adds two missing “GM at the table” affordances:

1. **Narrator / world events** — something happens in the room that is not any agent’s action (thunder, a door slam, a smell). Every agent should **see** it in passive vision and **remember** it like a witnessed observation.
2. **Visual tokens** — text chips work; optional `appearance` keys let realm-studio show emoji or simple icons without the sim caring about art assets.

**Multiplayer** (rooms, server-authoritative sessions, auth) is explicitly **deferred** to [LONG_TERM_GOALS.md](../LONG_TERM_GOALS.md). V0.4’s multi-area work may reuse snapshot shapes later, but we are not building netcode in 0.3.2.

## Area events — design (0.3.2a–b)

Today, observable text flows only from an **agent’s** `passive_result` after a turn (`broadcast_actor_turn`). There is no **area-wide** event source.

### Target behavior

| Actor | Example |
|-------|---------|
| GM clicks **Emit event…** in realm-studio | Modal: “Thunder rumbles overhead.” |
| Engine | `Session.emit_area_event(text)` — **does not consume a turn** |
| Passive vision (all agents) | Unchanged static entity lines only (events are memory-only) |
| Memory (all agents) | Ingest as witnessed observation (environment pseudo-actor) |
| Next **Run turn** | Event appears in **Memory:** section for each agent |

### Engine API (draft)

```python
result = session.emit_area_event("Thunder rumbles overhead.")
# SessionResult(ok, message)
```

Optional CLI parity:

```text
emit-event "Thunder rumbles overhead."
```

### Snapshot (draft)

```json
{
  "recent_events": [
    { "session_turn": 3, "text": "Thunder rumbles overhead." }
  ]
}
```

Ring buffer (e.g. last **5** events) on `Area`; included in `build_area_snapshot` for UI turn log / sidebar.

### HTTP (draft)

| Method | Path | Maps to |
|--------|------|---------|
| `POST` | `/api/event` | `{ "text": "..." }` → `session.emit_area_event` |

Response: `{ ok, message, snapshot? }` — same refresh pattern as `/api/command`.

### v1 rules

- Events are **room-wide** — no range/occlusion in 0.3.2
- Events **persist** in the area log until evicted by buffer size (not single-turn flash)
- **No** turn increment on emit (same class as area edits)
- **No** streaming or scheduled events

## Appearance — design (0.3.2c–d)

Visualization-only metadata. The simulation **ignores** `appearance` for movement, perception, and prompts unless we explicitly add that later.

### Engine (draft)

| Entity | Field | Example |
|--------|-------|---------|
| `Agent` | `appearance: str \| None` | `"🧙"`, `"token/explorer"`, `"#2d4a3e"` |
| `Object` | `appearance: str \| None` | `"⚽"`, `"token/ball"` |

- Exposed in `snapshot()` on agents/objects
- Set via `create-agent` / `edit-agent` / `create-object` / `edit-object` CLI flags (or realm-studio modals)
- Default `None` — realm-studio falls back to name text chip (current behavior)

### realm-studio (draft)

- If `appearance` is a single grapheme or short string → show as chip **icon** + name on hover
- If `appearance` looks like a CSS color → tile/chip accent (optional stretch)
- Asset paths like `token/ball` → reserved for a future asset folder; v1 may map a small built-in dictionary or show key as text

## 0.3.2a — Engine area events

**Status:** ✅ **Implemented** in code.

**Goal:** `Session.emit_area_event` so all agents perceive narrator/world text in passive vision and memory.

### Shipped

| Piece | Detail |
|-------|--------|
| **`AreaEventRecord` + ring buffer** | `Area.recent_events` (last 5) via `append_area_event` |
| **`Session.emit_area_event(text)`** | No turn increment; fans out to all agents |
| **`broadcast_area_event`** | Memory ingest via `WitnessedEvent` pseudo-actor (`__area__` / Environment) |
| **Passive vision** | Static entity lines only — no `passive_result` or area events (memory holds both) |
| **Memory formatting** | Area events render as plain `- {text}` (no actor/position suffix) |
| **CLI** | `emit-event "..."` via `run_command` and `ManualStepper` |
| **Snapshot** | `recent_events[]` with `{ session_turn, text }` |
| **Tests** | `tests/test_area_events.py` (8 tests) |

### Out (unchanged)

- Per-agent filtering (some agents deaf/blind)
- Scheduled / delayed events
- Event templates or effect hooks
