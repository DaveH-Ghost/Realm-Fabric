## 0.4.3a — Memory module on agent creation

**Status:** ✅ **Done**

The engine accepts memory module flags on `create-agent` (`memory`, `memory-window`, `memory-budget`, `memory-summary-*`). realm-studio exposes them in **Create agent**:

| Module | UI fields |
|--------|-----------|
| `recent_turns` | Turn window (`memory-window`, 1–100) |
| `salient_turns` | Character budget (`memory-budget`, 200–8000) |
| `rolling_summary` | Summary interval, max summary chars, detail tail |

- **`GET /api/memory-modules`** — catalog with defaults and validation ranges
- **Edit agent** — read-only memory module label (fixed at creation, same as CLI)
- Conditional form fields show/hide based on selected module

Memory module is still **create-only**; `edit-agent` cannot change it.

## 0.4.3b — Edit object/agent location panel

**Status:** ✅ **Done**

**Edit object** and **Edit agent** modals in realm-studio:

- **Location ▾** collapsible — area dropdown + X/Y (tile from right-click is no longer top-level fields)
- **`edit-object … area <area_id> pos x,y`** — engine finds objects in any area and moves them between areas
- **`edit-agent … area <area_id> pos x,y`** — same for agents (cross-area move via `Session.transfer_agent`)

Same CLI flags available outside realm-studio via `Session.run_command`.

## 0.4.3c — Multi-step witness broadcast

**Status:** ✅ **Done**

When an agent performs several observable steps in one compound turn, witnesses now receive **one memory event per step** (in execution order):

| Step kind | Witnessed when |
|-----------|----------------|
| move | Always (when movement has passive text) |
| speak | Always |
| emote | Always (per-observer phrasing, e.g. “at you”) |
| interact | Always |
| look | Only when no move / speak / emote / interact ran that turn |

Fixes speak + emote (or speak + interact) turns where only the turn action was visible to other agents.
