# Version 0.2.5 Changelog

**Purpose:**  
Living record of what ships in V0.2.5 and why — **not** a pre-implementation checklist. Each section corresponds to an incremental release slice (e.g. `0.2.5a`, `0.2.5b`) so we can implement and review agilely without a full spec up front.

**Baseline:** Builds on **V0.2** (`v0.2.0`) — coordinate move, compound turn *execution* (move → look → turn action), declarative object interact, effect registry, multi-agent manual stepper.

**Overall V0.2.5 focus:** Memory as a first-class, **pluggable** subsystem (memory modules per agent, eventual store/consolidation). See [ROADMAP.md](ROADMAP.md) for themes still planned after the slices below.

See also: [v0.2-implementation-readiness-checklist.md](v0.2-implementation-readiness-checklist.md) (V0.2 as-shipped reference), [LONG_TERM_GOALS.md](../LONG_TERM_GOALS.md).

## 0.2.5a — Single LLM compound turn

**Status:** ✅ **Implemented** in code (`pyproject.toml` → `0.2.5`). Tag not yet cut.

**Goal:** One LLM decision per agent turn so memory, logging, cost, and future memory modules align with “one turn = one call.” Compound *execution* stays the same (move first, then look, then speak/interact).

### Why this changed

V0.2 used **two LLM calls** per turn (navigation, then action with post-move vision). That made post-move look/interact lists accurate but conflicted with upcoming memory work (two prompts, two reasoning blocks, awkward turn boundaries). On a 5×5 grid, planning the full turn from **pre-move** vision is an acceptable tradeoff until we need richer post-move hints.

### Schema

- **Added:** `AgentCompoundTurn` in `src/llm/schemas.py` — single JSON for the full turn:
  - `reasoning`, `move_target`, `look_target`, `turn_action`, `target`, `action_name`, `content`, `confidence`, `emotion`
- **Removed from the LLM path:** `AgentNavigationTurn`, `AgentActionTurn` (design references under `docs/schemas/` not updated yet)
- Validation unchanged in spirit: coordinate `move_target`, speak limits (5 sentences / 500 chars), interact field rules, `ERR:*` patterns

### Prompt & client

- **Added:** `build_compound_prompt()` in `src/llm/prompt.py` — one prompt with character, room, passive vision, move rules, look targets, interact list, recent history, output format
- **Removed:** `build_navigation_prompt()`, `build_action_prompt()`
- **Added:** `get_compound_turn()` in `src/llm/client.py`
- **Removed:** `get_navigation_turn()`, `get_action_turn()`
- Instructions tell the model: move runs first; plan look/action for the position **after** the move using current vision; only target entities expected to be valid post-move
- Interact list in the prompt is still computed from the agent’s **current** position (pre-move)

### Runtime / stepper

- **`_run_llm_turn_for_agent`:** one call → parse → `run_compound_turn()`
- **`prompt` command:** prints the single compound prompt (no `nav` / `action` sub-commands)
- **Logging:** file log phase label `[compound]` (replaces `[nav]` + `[action]`)
- **Manual stepping unchanged:** `step-compound`, `step-nav`, `step-action` still work; parser returns one `AgentCompoundTurn`
- **Removed behavior:** partial turn when the *action* LLM failed after a successful nav call (impossible with one call; whole turn aborts on LLM/parse failure)

### TurnRecord

- **Changed:** `nav_reasoning` + `action_reasoning` → single `reasoning: str` on `TurnRecord` (`src/turn_record.py`)
- Look knowledge (`looked_at` / `ever_looked`) remains on the `Memory` facade (unchanged in 0.2.5a)

### Execution (unchanged)

`src/simulation.py` still runs:

1. Optional move (`execute_nav_phase`)
2. Optional look + turn action (`execute_action_phase`)

`passive_result` priority unchanged: **turn action > look > move**; mood suffix from the compound turn’s confidence/emotion.

Invalid look/interact after a successful move still records the move and fails the later step (same as V0.2 execution semantics).

### Breaking changes vs V0.2

| Area | V0.2 (`v0.2.0`) | V0.2.5a |
|------|------------------|---------|
| LLM calls per turn | 2 (nav + action) | 1 |
| Schemas | `AgentNavigationTurn` + `AgentActionTurn` | `AgentCompoundTurn` |
| Prompt builders | `build_navigation_prompt` + `build_action_prompt` | `build_compound_prompt` |
| `TurnRecord` reasoning | `nav_reasoning`, `action_reasoning` | `reasoning` |
| Action-phase vision in prompt | Post-move (second call) | Pre-move (plan for post-move) |

### Tests

- Suite updated for single-call path; **160 tests** green at implementation time
- `test_v0_2_ship.py` version check expects `0.2.5`; LLM logging test expects `phase="compound"`
- Removed test for “action LLM failed → partial turn with move only”

### Explicitly not in 0.2.5a

- `memory_modules/` directory or `"Memory:"` prompt section (planned **0.2.5b**)
- Database, goals, tiered policies, memory-manager LLM
- README / schema design docs / V0.2 checklist retro-updates (follow separately)

## 0.2.5b — Pluggable memory modules (`recent_turns`)

**Status:** ✅ **Implemented** in code (`pyproject.toml` → `0.2.5`). Tag not yet cut.

**Goal:** Prompt memory is a **pluggable module** per agent. Look knowledge stays on the `Memory` facade; the module owns turn history, witnessed other-agent actions, and the **`Memory:`** prompt block. Module id is chosen **only at agent creation** — never via `edit-agent`.

### Architecture

| Layer | Location | Responsibility |
|-------|----------|----------------|
| **Turn types** | `src/turn_record.py` | `TurnRecord`, `TurnStep`, `StepKind` |
| **Facade** | `src/memory.py` | Look knowledge + delegate to module (`record_turn`, `record_observation`, `render_prompt_block`) |
| **Protocol** | `src/memory_modules/base.py` | `MemoryModule`: `record_turn`, `record_observation`, `render`, `total_turns`, `stored_turns` |
| **Default module** | `src/memory_modules/recent_turns.py` | Last 10 own turns + witnessed actions between turns |
| **Registry** | `src/memory_modules/registry.py` | `create_module(id)`, `known_module_ids()`, `format_memory_modules_list()` |
| **Observations** | `src/observations.py` | After each turn commit, broadcast actor `passive_result` to observing agents |

### `recent_turns` module

- **Own turns:** up to **10** `TurnRecord`s (window); **`total_turns`** is monotonic (used for `TurnRecord.turn_number`).
- **Witnessed events:** when another agent completes a turn with a non-empty `passive_result` and the actor is in the observer’s passive vision, the observer’s module ingests a `WitnessedEvent` (session turn, actor id/name, text, position).
- **Render order:** for each own turn, optional **“Before turn N, you observed:”** block, then turn detail; pending witnesses since the last own turn appear as **“Since turn N, you observed:”** (or **“You observed:”** if none yet).
- **Prompt label:** `"Recent history:"` → **`"Memory:"`**; body from `agent.memory.render_prompt_block(agent, world)` (empty → `"No memories yet."`).

Passive vision still shows each agent’s **current** `passive_result` on their vision line; memory retains a **durable** witness history between the agent’s own turns.

### Module selection (create only)

- **Default:** `Memory()` → `recent_turns` (initial Explorer, agents without an explicit `memory` field).
- **Explicit at creation:**
  ```text
  create-agent name "Scribe" personality "Quiet." memory recent_turns at 2,2
  ```
- **Discovery:** `memory-modules` command; `agents` / `list` show `memory=<id>` per agent; `state` shows `Memory module: …`.
- **`edit-agent` does not support `memory` — by design.** Module choice is fixed for the agent’s lifetime in a session. Changing modules mid-run would need migration semantics; out of scope for V0.2.5b. To use a different module, create a new agent.

Unknown module ids fail at creation with a message listing `known_module_ids()`.

### Simulation hooks

- `commit_turn_record(..., world, session_turn=…)` → `memory.record_turn` then `broadcast_actor_turn`.
- `session_turn` from the stepper labels witness ordering; tests may omit it (falls back to actor `turn_number`).

### Breaking / visible changes vs pre-0.2.5b

| Area | Before | 0.2.5b |
|------|--------|--------|
| Prompt section | `Recent history:` | `Memory:` |
| Turn storage | Inline on `Memory._turns` | `RecentTurnsModule` via facade |
| Other agents’ actions | Passive vision only | Also ingested into observer memory when witnessed |
| `turn_count` | Len of stored turns (capped at 10) | Module `total_turns` (uncapped) |
| Agent listing | id, name, position | Includes `memory=<module_id>` |

### Tests

- `tests/test_memory_modules.py` — module render/record, window cap, witness gaps, multi-agent ingest, create-agent `memory` field, unknown module rejection.
- Existing suite updated (**176 tests** green at implementation time).

### Explicitly not in 0.2.5b

- Salience-based second module (see **0.2.5c**)
- `edit-agent memory …` (intentionally unsupported, ever)
- Per-module CLI config (e.g. `window` on create-agent); `Memory(module_id=…, **config)` works in code for tests/future use
- Database, summarization LLM, tiered policies
- README / V0.2 checklist retro-updates (optional follow-up)

## 0.2.5c — Salient turns memory module (`salient_turns`)

**Status:** ✅ **Implemented** in code (`pyproject.toml` → `0.2.5`). Tag not yet cut.

**Goal:** A second memory module that uses the **same ingest** as `recent_turns` but applies **retention and render policy** — keep turns that “mattered,” drop low-signal move-only history, and cap the **`Memory:`** body with a per-agent **character budget** set at creation.

Demonstrates module **policy** (not just formatting or window size) while `recent_turns` remains the safe default fallback.

### Ingest (unchanged from 0.2.5b)

- `record_turn` / `record_observation` — same `TurnRecord`, `WitnessedEvent`, and witness-on-commit flow as `recent_turns`.
- Shared render helpers in `src/memory_modules/formatting.py` (extracted from `recent_turns`).

### Salience scoring (storage retention)

Each own turn is scored when recorded:

| Signal | Score |
|--------|------:|
| speak or interact step | +3 |
| look step | +2 |
| non-empty witnesses before this turn | +1 |
| move-only, no witnesses | 0 |

**Storage window:** up to **50** turns in memory (configurable in factory). When over capacity, evict **lowest score** first; **recency floor** (last **2** turns) is never evicted from storage.

`total_turns` remains monotonic (same as `recent_turns`).

### Render (character budget)

- **`char_budget`** caps the rendered `Memory:` body (default **2500** characters).
- Selection order: **recency-floor blocks first** (newest among protected), then remaining blocks by **salience** (newest on ties).
- Output stays **chronological** among selected blocks.
- When older blocks are skipped: **`…earlier memories omitted.`** at the top.
- Pending witnesses since the last own turn are treated as recency-priority content.

Budget applies to the **Memory:** section only, not the full compound prompt.

### Create-agent wiring

```text
create-agent name "Minion" personality "..." memory salient_turns memory-budget 1200 at 0,0
```

| Field | Rule |
|-------|------|
| `memory salient_turns` | Use salient module (default budget 2500) |
| `memory-budget N` | **200–8000**; valid only with `salient_turns` |
| `memory-budget N` alone | Implies `memory salient_turns` |
| `memory recent_turns memory-budget …` | **Rejected** |
| omitted `memory` | Still **`recent_turns`** (unchanged default) |

**`edit-agent` does not support memory or budget** — same fixed-at-creation policy as 0.2.5b.

### Discovery / visibility

- **`memory-modules`** lists `salient_turns` with budget note.
- **`agents` / `list`:** `memory=salient_turns budget=2500` (via `format_memory_module_label`).
- **`state`:** module label + `Memory char budget:` for salient agents.

### Registry

```python
"salient_turns": lambda **cfg: SalientTurnsModule(
    char_budget=..., storage_window=..., recency_floor=...
)
```

`create_module("recent_turns", char_budget=…)` is rejected at the registry.

### Comparison: `recent_turns` vs `salient_turns`

| | `recent_turns` | `salient_turns` |
|--|----------------|-----------------|
| Retention | Last **10** turns (FIFO) | Up to **50**; salience eviction |
| Render limit | All stored turns | **char_budget** |
| Move-only quiet turns | Kept until aged out | Dropped first when storage/render tight |
| Witness ingest | Yes | Yes |
| Create config | `memory recent_turns` | `memory salient_turns` + optional `memory-budget` |

### Tests

- `tests/test_salient_turns.py` — scoring, storage eviction, recency floor, char-budget render, omission line, witnesses, create-agent validation, registry rules.
- `tests/test_memory_modules.py` updated for both module ids in registry listing.
- **192 tests** green at implementation time.

### Explicitly not in 0.2.5c

- LLM summarization / memory-manager consolidation
- `edit-agent` memory or budget changes
- Per-agent `storage_window` / `recency_floor` on CLI (factory kwargs only in code)
- README / V0.2 checklist retro-updates (optional follow-up)

**Note:** Turn render format and salient scoring were **refined in 0.2.5d** (condensed `Result:` output, step-level salience). See below for current behavior.

## 0.2.5d — Condensed memory, fragment salience, look-list fixes

**Status:** ✅ **Implemented** in code (`pyproject.toml` → `0.2.5`). Tag not yet cut.

**Goal:** Cut redundant token weight in **`Memory:`**, make **`salient_turns`** retention match compound-turn structure (speak/interact ≫ look ≫ move), and align the compound prompt’s look guidance with what the agent actually sees.

### Why this changed

Compound turns store move, look, and speak/interact as separate steps, but memory was rendering **every step plus the full composite `Result:`** — often tripling the same text. Turn-level salience also scored almost every social turn highly (+ witnesses on most turns), so **`salient_turns`** did little useful pruning. The prompt referred to “passive vision” in look rules while the vision block is labeled **`Current situation:`**, and **`You can look at:`** listed entities the agent had already examined.

### Shared turn format (`formatting.py`) — both modules

Own turns now render as:

```text
Turn N:
Reasoning: …
Result: …
```

| Rule | Detail |
|------|--------|
| **No step list** | Removed per-step `- move → …` lines; no duplicate composite under `Result:` |
| **Reasoning window** | **`Reasoning:`** only for the **newest 3** stored own turns (`REASONING_WINDOW = 3`) |
| **`recent_turns`** | **`Result:`** is always the full composite `turn.result` |
| **`salient_turns`** | **`Result:`** built from **selected step fragments** (see below) |

Witness blocks (**“Before turn N, you observed:”** / **“Since turn N, you observed:”**) unchanged in shape; full witness text still shown for **`recent_turns`**. **`salient_turns`** prioritizes witness blocks at speak-tier salience in budget selection (no protocol changes).

### Salient — step-level salience

Each **step kind** has a fixed weight (peak weight used for storage eviction):

| Step kind | Salience |
|-----------|----------:|
| **speak** | 10 |
| **interact** | 7 |
| **look** | 3 |
| **move** | 1 |
| **witnessed event block** | 10 (same tier as speak; salient module only) |

**Storage eviction** (50-turn cap, recency floor 2): evict the stored turn with the **lowest peak step salience** (move-only turns go first). Replaces turn-level scoring (+3/+2/+1 witness) from initial 0.2.5c.

**Render — fragment trimming by age:**

| Turn age | `Result:` contents |
|----------|-------------------|
| **Recency floor** (last **2** stored turns) | All step results (move, look, speak/interact) |
| **Older turns** | **speak** and **interact** fragments only; move and look dropped |
| **Move-only, outside floor** | Turn omitted from render entirely |

**Char budget** selection unchanged in spirit: recency-floor blocks first, then highest salience; chronological output; **`…earlier memories omitted.`** when older content is skipped.

### Compound prompt — look guidance & list

| Area | Change |
|------|--------|
| **Important rules** | `look: optional; a list of objects you can look at will be provided.` (replaces passive-vision wording) |
| **`You can look at:`** | Only entities whose **`Current situation:`** line shows **`[?]`** (never examined or stale **`[?] [changed]`**). Already-examined objects/agents and passive-only objects (no hidden detail) are omitted. |
| **Execution** | `perform_look` still accepts any visible target if the model sends one; only the **prompt list** is filtered |

Vision block label remains **`Current situation:`** (room + passive vision lines).

### Tests

- `tests/test_memory_modules.py` — condensed format, reasoning window.
- `tests/test_salient_turns.py` — `storage_salience`, fragment trimming, recency floor keeps full compound `Result:`.
- `tests/test_perception.py` — filtered look targets, prompt rule text.
- **200 tests** green at implementation time.
